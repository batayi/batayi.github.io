Web GIS
Stack: PostgreSQL/PostGIS, GeoServer, Leaflet/JavaScript
Focus: Serving raster & vector data through OGC services to web clients

Overview

I developed Web GIS applications that expose spatial data through APIs and map services, allowing users to explore environmental and socio-economic datasets interactively in a browser.

Architecture

  • PostgreSQL/PostGIS as the main spatial database.
  • GeoServer configured to serve WMS/WFS/WCS layers.
  • Leaflet-based frontends consuming OGC services and custom REST endpoints.

Features

  • Layer switching and styling (base maps, thematic layers, overlays).
  • Attribute inspection and basic querying (identify, filter by attributes).
  • Time-sliders or controls for multi-year datasets where relevant.
  • Custom symbology for highlighting forest change, agricultural intensity, or other indicators.

Example Use Cases

Web GIS dashboards for:

  • Visualising forest change and deforestation risk layers.
  • Exploring agricultural adoption and yield patterns.
  • Serving internal mapping products to non-GIS users.
